## Requirements

- Scaffold Next.js app router project in repository root.
- Add Supabase integration utilities for browser/server/admin contexts.
- Add server-side invite verification and reCAPTCHA verification on signup.
- Add initial SQL schema with core tables and moderation status fields.
- Add starter pages for feed, log detail, profile, leaderboard, signup, and admin.

## Implementation Plan

- Add environment handling and validation helpers.
- Add domain types and initial mock data for UI progress before production data plumbing.
- Add secure signup route handler validating payload, invite code, and CAPTCHA token.
- Add Supabase schema SQL for invites, profiles, logs, comments, votes, and flags.
- Add route/page shells to support feature-by-feature iteration.

## Test Plan

- Run lint and type checks.
- Test signup API with invalid payload, invalid invite, invalid CAPTCHA, and valid payload.
- Validate that pages render with mock data when no Supabase env vars are set.

## Verification

- `npm run lint` passes.
- `npm run build` passes and all implemented routes compile successfully.
- Signup API includes payload validation, invite validation, reCAPTCHA verification, and rollback-on-failure provisioning logic.
