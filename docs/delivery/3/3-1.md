## Requirements

- Provide a login page with email and password inputs.
- Use Supabase Auth to sign in with email/password.
- Show clear error feedback for invalid credentials or unconfirmed email.
- Redirect to the feed on successful login.
- Expose a Log In link in the header for signed-out users.

## Implementation Plan

1. Create a login page route that mirrors the signup layout and messaging.
2. Build a LoginForm client component with:
   - Email and password inputs.
   - Inline validation to prevent empty submissions.
   - Submit handler that calls `signInWithPassword` via the Supabase browser client.
3. Handle responses:
   - Show user-friendly error messages on failure.
   - Redirect to the feed on success.
4. Update header navigation to include a Log In link when no active session is detected.
5. Validate behavior with a signed-out and signed-in user.

## Test Plan

- Run `npm run lint`.
- Manually test login with valid credentials.
- Manually test login with invalid credentials and confirm errors render clearly.
- Verify the Log In link appears when signed out and disappears after sign-in.
