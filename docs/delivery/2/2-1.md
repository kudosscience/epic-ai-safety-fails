## Requirements

- Select the lowest-cost hosting option that supports Next.js App Router and API routes.
- Specify environment variable configuration steps and secrets handling.
- Include reCAPTCHA domain allowlist updates for the test domain.
- Define validation checks for signup, invite flow, and admin gating.

## Implementation Plan

1. Hosting choice (lowest-cost)
   - Use Vercel free tier for the public test deployment. It supports Next.js App Router and API routes with $0 cost.
   - Record any current free-tier limits in the plan and link to Vercel pricing for verification.
2. Prepare the Supabase test project
   - Create or reuse a Supabase project dedicated to the public test environment.
   - Apply the baseline schema from [supabase/schema.sql](supabase/schema.sql) in the Supabase SQL editor.
   - Create at least one invite code entry consistent with the schema so testers can sign up.
   - Confirm RLS policies align with expected read/write access for public and authenticated users.
   - Reference [docs/delivery/1/1-1-supabase-guide.md](docs/delivery/1/1-1-supabase-guide.md) for client setup patterns.
3. Configure reCAPTCHA v3 for the test domain
   - Create v3 keys for the test environment.
   - Add the Vercel test domain (and any custom domain) to the reCAPTCHA allowlist.
   - Reference [docs/delivery/1/1-1-recaptcha-guide.md](docs/delivery/1/1-1-recaptcha-guide.md) for token rules and verification details.
4. Configure the Vercel project (manual steps)
   - Connect Vercel to the GitHub repo and import the project.
   - Select the main branch for production deployments.
   - Add environment variables using [.env.example](.env.example) as the source of required keys:
     - `NEXT_PUBLIC_SUPABASE_URL`
     - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
     - `SUPABASE_SERVICE_ROLE_KEY`
     - `NEXT_PUBLIC_RECAPTCHA_SITE_KEY`
     - `RECAPTCHA_SECRET_KEY`
   - Set these variables for both Production and Preview environments as needed.
   - Trigger the initial deployment and record the test URL.
5. Post-deploy validation checklist
   - Confirm the landing page and filters render with mock data or live data.
   - Confirm signup rejects invalid payloads with clear errors.
   - Confirm signup accepts a valid invite code and reCAPTCHA token.
   - Confirm admin route redirects unauthenticated users and blocks non-admins.
   - Confirm moderation states render as under-review without hiding content.
6. Rollback and cleanup
   - If the deployment must be disabled, remove environment variables or unpublish the project.
   - Rotate Supabase and reCAPTCHA keys if any secret exposure is suspected.
