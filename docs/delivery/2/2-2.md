## Requirements

- Identify the exact request payload and schema mismatch causing the error.
- Confirm whether the error is caused by client input, reCAPTCHA token, or server validation.
- Define a minimal fix path without scope creep.

## Implementation Plan

1. Reproduce
   - Attempt signup in the test environment and capture the request body in DevTools.
   - Note all fields (`email`, `password`, `username`, `fullName`, `inviteCode`, `recaptchaToken`).
2. Validate schema alignment
   - Compare the payload against `signupRequestSchema` in the signup route.
   - Check `email` format, `username` regex, invite code length/trim rules, and recaptcha token length.
3. Check reCAPTCHA flow
   - Ensure `recaptchaToken` is present and generated after the script is ready.
   - Confirm the expected action value matches `signup`.
4. Review server-side response
   - Inspect API response and logs for validation errors.
   - Confirm the error path (`Invalid signup payload`) is reached due to schema parse failures.
5. Remediation plan
   - If client formatting is wrong, adjust input normalization or field names.
   - If schema is too strict for the UI, update schema and document the reason.
   - If reCAPTCHA is missing, verify site key configuration and script readiness.

## Test Plan

- Validate the signup request body matches the schema after any planned adjustments.
- Confirm the error message is cleared for valid payloads.
- Ensure invalid payloads still return 400 with a clear error message.

## Verification

- Root cause of the payload error is documented with evidence.
- Proposed fix is scoped and does not alter unrelated behaviors.
